# Contributing

ForkLedger is open to improvements in architecture, retrieval, evaluation, documentation, and domain integrations.

## Good first contributions

- add more realistic scoring functions
- improve the policy distillation layer
- add benchmarks and synthetic datasets
- add richer explainability output
- integrate with agent frameworks
- add web or notebook visualizations

## Development

```bash
pip install -e .
python -m pytest
```

## Standards

- keep APIs simple and inspectable
- prefer explicit data structures over hidden magic
- document design decisions
- add tests when changing behavior
